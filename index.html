Every course and question you add saves directly to your Supabase database. Anyone who visits your site — on any device — sees those changes immediately. No re-uploading files needed.

Import from backup
Upload a previously exported backup file. This adds those courses/questions into your live database (it won't delete anything already there).
⚠️ Changing your admin password
Open index.html, search for const ADMIN_PASSWORD, and replace the value with your own. Save and re-upload to GitHub.

Note: this is a simple client-side lock — fine for keeping casual visitors out, but technically visible to anyone who views your page's source code. Don't reuse an important password here.
Reconnecting on a new device
The database connection (URL + key) is remembered per browser. If you open the Admin panel on a new computer or browser, you'll be asked to paste your Supabase URL and anon key again — find them in Supabase under Settings → API.
